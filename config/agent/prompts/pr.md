Look at the changes on this branch vs `main` (`git log main..HEAD --oneline` and `git diff main...HEAD`).

Write a PR description with:
- **Title**: concise, conventional commit style
- **Summary**: 2-3 sentences on what and why
- **Changes**: bullet list of key changes
- **Testing**: how this was or should be tested

Keep it concise. Don't list every file changed — summarize the intent.
